Qualifying technologies & the invariants that make Relay Relay
Slack AIcapabilities
Ask-Relay assistant — suggested prompts, "thinking" status, answers grounded in the PII-free ledger with cited permalinks; refuses out-of-scope & emergency-dispatch questions.
Real-Time SearchRTS API
assistant.search.context client (throttled + retrying) grounds Ask-Relay in live field context; lights up with a user token, else degrades to ledger-only. Results cited, never persisted.
MCPintegration
Relay is exposed as a read-only MCP server (search_needs · get_need · get_sitrep) so Claude Desktop & other agents query live operations — over the same PII-free projections.
Append-only truth

No UPDATE status anywhere; state is a replay of events. The audit trail is the product.

Humans in command

Confirm · assign · merge · sign-off · close all require a human actor — the engine rejects, it doesn't trust callers.

PII, minimized

Beneficiary contact only in the encrypted vault; ledger & LLM inputs are zero-copy. Reveal writes an audit event.

Numbers never guessed

Reports render {{stat:*}} tokens; a validator rejects any figure not backed by the ledger — proven to fall back to template.

Hosting: Fly.io — Docker machine + self-hosted Postgres + Redis · auto-HTTPS on *.fly.dev · always-on so the drift tick never sleeps. Runs fully hermetically with zero infra for tests & the offline demo.
