You own **W6 — the customer trust page** for Kept, the retention-weapon wow: a per-account page showing commitments Kept / In-progress / Verifying / At-risk. Read `CLAUDE.md`; invariants #5 (audience policy) + #4 (tenant isolation) are your mandate. This is customer-facing, so it MUST be safe by construction by reusing the existing D1 gate — you are adding a new *surface* over the *same* `src/policy/audience.ts` sanitizer, not a new policy.

Scope:
- Route `GET /trust/:token` as a Bolt `customRoute` (served from the one HTTPS listener that W2 sets up). `:token` = an opaque, unguessable per-(team, customer) capability stored in a `trust_links` table (random token, revocable) — the token IS the authorization; no login.
- `buildTrustView(teamId, customer)` in `src/app/*`: scoped `listObligations(teamId)` (never unscoped) filtered to that customer, bucketed Kept / In progress / Verifying / At-risk. **Every displayed string runs through `sanitizeForAudience(evidence, "SHARED_CUSTOMER_CHANNEL")` + `detectLeaks`**; if a `outcome` trips the detector, render a generic label ("Commitment #N"). Show: sanitized outcome, due date, status bucket, "kept on <date>", counts. NEVER render ticket/PR/release refs, internal sources, evidence internals, reconciliation details, or any other customer/team.
- Response headers: `X-Robots-Tag: noindex`, `Cache-Control: private, no-store`; rate-limit; a "generated by Kept" footer. A small branded HTML page matching the Vercel design tokens (paper/teal, Fraunces/IBM Plex).
- Add a way to mint/revoke a trust link (e.g. an Assistant/`/kept` action or an internal admin route) scoped to the acting team.

Acceptance: `tests/trustPage.test.ts` — a leaky outcome ("done PROJ-118") renders sanitized (never the raw ref); a token for (teamA, Acme) cannot read (teamB, *) or (teamA, Globex); an obligation with internal-only evidence shows no internal source. Typecheck + suite green.
